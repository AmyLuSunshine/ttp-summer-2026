# Glossary

Quick reference for terms you'll encounter throughout the program. If something isn't here, Google it.

---

**API (Application Programming Interface)** — A set of rules that lets two applications talk to each other and share data.

**API Key** — A secret code used to identify your app when making requests to a third-party API.

**Array** — An ordered list of values stored in a single variable, accessed by index (starting at 0).

**Arrow Function** — A shorter way to write a JavaScript function using `=>` syntax.

**Async / Await** — JavaScript keywords that let you write asynchronous code that reads like synchronous code.

**Attribute** — Extra information added to an HTML element, like `src` on an image or `href` on a link.

**Auth0** — A third-party service that handles user authentication so you don't have to build it from scratch.

**Authentication** — The process of verifying who a user is (e.g., logging in with a username and password).

**Authorization** — The process of determining what a logged-in user is allowed to do or access.

---

**Backend** — The server-side of a web application that handles data, business logic, and database communication.

**Bash** — A command-line shell and scripting language used in terminals to run commands.

**Branch** — A separate line of development in Git that lets you work on a feature without affecting the main codebase.

**Bug** — An error or flaw in code that causes unexpected behavior.

**Build** — The process of converting source code into an optimized, production-ready version of your app.

---

**Callback** — A function passed as an argument to another function, to be called after an operation completes.

**CDN (Content Delivery Network)** — A network of servers that delivers assets (images, CSS, JS) quickly based on the user's location.

**CLI (Command Line Interface)** — A text-based interface where you type commands to interact with your computer.

**Client** — The browser or frontend app that sends requests to a server.

**Client-Side Routing** — Navigation between pages in a React app handled in the browser, without a full page reload.

**Clone** — To copy a remote Git repository onto your local machine.

**Closure** — A function that retains access to variables from its outer scope even after that scope has finished running.

**Column** — A single field in a database table (e.g., `name`, `email`, `created_at`).

**Commit** — A saved snapshot of your code changes in Git, with a descriptive message.

**Component** — A reusable, self-contained piece of UI in React that can accept props and manage its own state.

**Console** — A tool in the browser or terminal for logging messages, inspecting values, and debugging.

**Context API** — A built-in React feature for sharing data across components without passing props at every level.

**Cookie** — A small piece of data stored in the browser, often used to maintain session information.

**CRUD** — Stands for Create, Read, Update, Delete — the four basic operations for managing data.

**CSS (Cascading Style Sheets)** — The language used to style and visually arrange HTML elements on a page.

---

**Database** — An organized collection of data that can be stored, queried, and updated.

**Debugging** — The process of finding and fixing errors in your code.

**Deployment** — Publishing your application to a hosting platform so others can access it on the internet.

**Destructuring** — A JavaScript shorthand for extracting values from arrays or objects into individual variables.

**DOM (Document Object Model)** — A tree-like representation of an HTML page that JavaScript can read and manipulate.

---

**Element** — A single HTML tag and its content (e.g., `<p>Hello</p>`).

**Endpoint** — A specific URL in an API that handles a particular type of request (e.g., `GET /api/users`).

**Environment Variable** — A value stored outside your code (in a `.env` file) used for secrets like API keys or database URLs.

**ES6 (ECMAScript 2015)** — A major update to JavaScript that introduced arrow functions, `let`/`const`, template literals, and more.

**Event** — An action that occurs in the browser, like a button click, key press, or page load.

**Event Listener** — A function that waits for a specific event and runs in response when it fires.

**Express.js** — A minimal Node.js framework for building web servers and APIs.

---

**Flexbox** — A CSS layout model for aligning and distributing items in a row or column.

**Foreign Key** — A column in a database table that references the primary key of another table, linking the two.

**Fork** — To copy someone else's GitHub repository to your own account so you can modify it independently.

**Framework** — A collection of pre-written code that gives your project structure and built-in tools.

**Frontend** — The client-side of a web app — everything the user sees and interacts with in the browser.

**Full Stack** — Refers to both the frontend and backend of a web application together.

**Function** — A reusable block of code that performs a specific task when called.

---

**Git** — A version control system that tracks changes to your code over time.

**GitHub** — A cloud platform for hosting Git repositories and collaborating with others.

**Grid (CSS Grid)** — A CSS layout system for building two-dimensional layouts using rows and columns.

---

**Higher-Order Function** — A function that takes another function as an argument or returns a function as its result.

**Hook** — A special React function (like `useState` or `useEffect`) that lets you use React features inside functional components.

**HTML (HyperText Markup Language)** — The standard language for structuring content on the web.

**HTTP (HyperText Transfer Protocol)** — The protocol used to transfer data between a client and a server on the web.

**HTTPS** — A secure version of HTTP that encrypts data in transit.

---

**ID** — A unique identifier for a single HTML element, used in CSS with `#` and in JavaScript to select specific elements.

**IDE (Integrated Development Environment)** — A code editor (like VS Code) with built-in tools for writing, running, and debugging code.

**Import / Export** — JavaScript syntax for sharing code between files using `import` and `export` keywords.

---

**JavaScript** — The programming language of the web, used to add interactivity and logic to web pages.

**JOIN** — A SQL operation that combines rows from two or more tables based on a shared column.

**JSON (JavaScript Object Notation)** — A lightweight text format for storing and exchanging data, structured as key-value pairs.

**JSX** — A syntax extension used in React that looks like HTML and lets you describe UI components inside JavaScript.

**JWT (JSON Web Token)** — A compact encoded token used to securely transmit user identity between a client and server.

---

**Library** — Pre-written code you can pull into your project to add specific functionality without building it yourself.

**Localhost** — The address (`localhost` or `127.0.0.1`) that points to your own machine, used when running a server during development.

---

**Merge** — Combining changes from one Git branch into another.

**Merge Conflict** — A situation in Git where two branches have conflicting changes that must be manually resolved.

**Middleware** — A function in Express that runs between receiving a request and sending a response, used for things like logging or authentication.

**Migration** — A file that defines changes to your database schema in a controlled, versioned way.

**Model** — In Sequelize, a JavaScript class that represents a database table and its columns.

**Module** — A self-contained JavaScript file that exports code for use in other files.

---

**Neon** — A cloud-hosted PostgreSQL service used to deploy databases to production.

**Node.js** — A JavaScript runtime that lets you run JavaScript on the server, outside of the browser.

**npm (Node Package Manager)** — A tool for installing and managing JavaScript packages and libraries.

---

**Object** — A JavaScript data type that stores related data as key-value pairs.

**OAuth** — An open standard that lets users grant third-party apps limited access to their accounts without sharing passwords.

**ORM (Object-Relational Mapper)** — A tool (like Sequelize) that lets you interact with a database using JavaScript instead of raw SQL.

---

**package.json** — A configuration file in a Node.js project that lists dependencies, scripts, and project metadata.

**PERN Stack** — A set of technologies — PostgreSQL, Express.js, React, Node.js — used together to build full-stack web apps.

**Port** — A number that identifies a specific channel a server listens on (e.g., `:3000` for Express, `:5432` for PostgreSQL).

**PostgreSQL** — An open-source relational database that stores data in structured tables and uses SQL to query it.

**Primary Key** — A unique identifier for each row in a database table, ensuring no two rows are identical.

**Promise** — A JavaScript object representing the eventual result (or failure) of an asynchronous operation.

**Props (Properties)** — Data passed from a parent component to a child component in React.

**Pull** — A Git command that downloads and integrates the latest changes from a remote repository.

**Pull Request (PR)** — A GitHub feature for proposing changes from one branch to another and requesting a code review.

**Push** — A Git command that uploads your local commits to a remote repository.

---

**Query** — A request sent to a database to retrieve, insert, update, or delete data.

---

**React** — A JavaScript library for building dynamic user interfaces using reusable components.

**React Router** — A library that adds client-side routing to React apps.

**Re-render** — When React updates and redraws a component because its state or props changed.

**Repository (Repo)** — A folder tracked by Git that contains all your project files and their full version history.

**Responsive Design** — A design approach that makes a web page look good on screens of all sizes.

**REST API** — A web API that uses standard HTTP methods (GET, POST, PUT, DELETE) to perform CRUD operations on resources.

**Route** — In Express.js, a definition of how the server responds to a specific URL and HTTP method.

**Row / Record** — A single entry in a database table, representing one item (e.g., one user, one post).

---

**Schema** — The structure of a database — its tables, columns, data types, and relationships.

**Selector** — A pattern in CSS used to target specific HTML elements for styling (e.g., `.class`, `#id`, `tag`).

**Sequelize** — A Node.js ORM that lets you interact with a PostgreSQL database using JavaScript instead of raw SQL.

**Server** — A program (or machine) that receives requests from clients and sends back responses.

**Session** — A temporary record that tracks a logged-in user across multiple requests.

**Single Page Application (SPA)** — A web app that loads once and dynamically updates content without full page reloads.

**SQL (Structured Query Language)** — The language used to communicate with relational databases like PostgreSQL.

**Spread Operator** — A JavaScript syntax (`...`) used to expand or copy the contents of an array or object.

**Staging Area** — In Git, the intermediate step where you prepare changes before committing them.

**State** — Data stored inside a React component that, when updated, causes the component to re-render.

**Status Code** — A three-digit number in an HTTP response indicating the result (e.g., `200` OK, `404` Not Found, `500` Server Error).

---

**Table** — A structured set of data in a database organized into rows and columns, similar to a spreadsheet.

**Template Literal** — A JavaScript string wrapped in backticks (`` ` ``) that can embed variables using `${}`.

**Terminal** — A text-based interface for running commands on your computer (also called the command line or shell).

---

**URL (Uniform Resource Locator)** — The web address used to access a specific resource on the internet.

**useContext** — A React hook that lets a component read values from a Context without prop drilling.

**useEffect** — A React hook that runs side effects (like fetching data) after a component renders.

**useState** — A React hook that adds state to a functional component and triggers a re-render when the value changes.

---

**Variable** — A named container for storing a value in code (declared with `let`, `const`, or `var` in JavaScript).

**Vercel** — A cloud platform for deploying frontend and full-stack JavaScript applications.

**Viewport** — The visible area of a web page within the browser window.

**Virtual DOM** — A lightweight in-memory copy of the real DOM that React uses to efficiently calculate and apply UI updates.

**Vite** — A fast build tool and development server used to scaffold and bundle modern JavaScript projects.
